## Summary

`rocket` is a new, standalone repository that hosts a small, distinctive
component library distributed as a **shadcn registry**. It is a single Next.js
(App Router) application that serves two roles at once:

1. A **docs / playground site** â€” a gallery that previews each component live,
   with copyable install commands and usage.
2. A **shadcn registry** â€” static JSON descriptors served at `/r/<name>.json`,
   installable in any project via `npx shadcn@latest add <url>`.

The first release ships exactly **one** component â€” `Timeline` â€” but builds the
full end-to-end pipeline (authoring â†’ `shadcn build` â†’ serving â†’ install â†’ docs
preview) so that adding further components later is mechanical.

This work is split out of `gozturk.dev`. `gozturk.dev` keeps its own JSX
playground; `rocket` is the clean, TypeScript home for shareable components.

## Goals

- Anyone can run `npx shadcn@latest add https://rocket.gozturk.dev/r/timeline.json`
  and get a working `Timeline` in **their own** project, themed by **their own**
  shadcn tokens.
- TypeScript (strict) throughout; proper exported types for component props.
- Live docs site that doubles as the registry host.
- Tooling consistent with `gozturk.dev` habits (Biome, pnpm, Node 22).

## Non-Goals (this release)

- More than one component (Timeline only). Infra is built to add more, but none
  are authored now.
- A published npm package (distribution is the shadcn copy-in model, not npm).
- A custom "rocket" brand theme preset (`registry:theme`). Deferred; components
  use standard shadcn tokens so they inherit the consumer's theme.
- Versioning / changelog automation, search, MDX docs authoring system.

- **Next.js** (latest stable, App Router) + React.

### Registry mechanics

- **`registry.json`** declares the registry name, homepage, and an `items` array.
  The one item:

  ```jsonc
  {
    "$schema": "https://ui.shadcn.com/schema/registry.json",
    "name": "rocket",
    "homepage": "https://rocket.gozturk.dev",
    "items": [
      {
        "name": "timeline",
        "type": "registry:ui",
        "title": "Timeline",
        "description": "A nested, collapsible event timeline with one continuous connector line.",
        "dependencies": ["lucide-react", "motion"],
        "registryDependencies": [],
        "files": [
          { "path": "components/ui/timeline.tsx", "type": "registry:ui" }
        ]
      }
    ]
  }
  ```

- **Build:** `pnpm registry:build` (wraps `shadcn build`) emits
  `public/r/timeline.json` (and any index). Generated artifacts are committed or
  built in CI â€” the implementation plan picks one (default: build in `vercel build`
  via a `prebuild`/`build` script so JSON is always fresh).

- **Serve:** Next serves `public/r/timeline.json` at `/r/timeline.json`.

- **Install (consumer side):**
  `npx shadcn@latest add https://rocket.gozturk.dev/r/timeline.json`
  pulls `timeline.tsx` into the consumer's `components/ui`, installs
  `lucide-react` + `motion`, and the component renders against the consumer's
  shadcn tokens.

### Timeline component (port to TS + portable tokens)

Source of truth lives in `rocket`; ported from the current `gozturk.dev`
`components/ui/timeline.jsx` (the **single-SVG-path connector** version â€” keep
that fix verbatim).

Changes:

- `.jsx` â†’ `.tsx`; add exported types:
  - `TimelineItem` â€” `{ id?; title?; time?; description?; status?: TimelineStatus; children?: TimelineItem[]; defaultOpen?: boolean }`
  - `TimelineStatus` â€” `'error' | 'success' | 'warning' | 'pending' | 'info'`
  - `TimelineProps` â€” `{ items?: TimelineItem[]; className?: string }`
- Token mapping (`p3-*` â†’ shadcn):
  - `text-p3-text` â†’ `text-foreground`
  - `text-p3-text-dark` (dark variant) â†’ dropped (shadcn tokens are theme-aware;
    no separate `-dark` token needed)
  - `border-p3-border` / `dark:border-p3-border-dark` â†’ `border-border`
  - `bg-p3-background-light` / `dark:bg-p3-background-dark` â†’ `bg-background`
    (toggle surface) â€” verify visually; may use `bg-card`/`bg-muted`
  - `text-neutral-400/500` â†’ `text-muted-foreground`
  - Connector stroke: `text-p3-border dark:text-p3-border-dark` â†’ `text-border`
    (SVG uses `stroke="currentColor"`).
  - Status colors (red/amber/green with `dark:` variants) â†’ **kept as-is**; they

### Hosting

- Deploy on Vercel. `pnpm build` runs `shadcn build` first (so `/r/*.json` is
  generated for the deployed site).
- `NEXT_PUBLIC_REGISTRY_URL` set per environment; the install command in docs and
  any absolute URLs derive from it. Production defaults to `https://rocket.gozturk.dev`.

## Data Flow

1. **Author** edits `components/ui/timeline.tsx` and `registry.json`.
2. **Build** (`shadcn build`) reads `registry.json` + source files â†’ writes
   `public/r/timeline.json` (self-contained: file contents + deps + metadata).
3. **Serve**: Vercel serves the Next app; `/r/timeline.json` is the registry item;
   `/` is the docs gallery rendering the live `Timeline`.
4. **Consume**: external dev runs `shadcn add <url>` â†’ CLI fetches JSON, writes
   files, installs deps into their project.

## Error Handling / Edge Cases

- **Stale registry JSON**: prevented by generating during `build` (no manual
  step to forget). Local `pnpm registry:build` documented for quick checks.
- **Wrong install URL in docs**: avoided via `NEXT_PUBLIC_REGISTRY_URL`; never
  hardcode the domain in components/pages.
- **Token mismatch on consume**: components rely only on standard shadcn tokens,
  which exist in any `shadcn init` project. Status accent colors use raw Tailwind
  palette (always present). No `p3-*` leakage â€” enforced by a grep check in
  verification.
- **Timeline measurement before paint**: `useLayoutEffect` + `ResizeObserver`
  already handle layout/animation; carried over unchanged.

## Testing / Verification

No unit test runner is mandated (matches `gozturk.dev`). Verification is
behavioral + static:

- `pnpm build` succeeds and emits `public/r/timeline.json`.
- `pnpm lint` (Biome) passes; `tsc --noEmit` passes.
- `grep -r "p3-" components/` returns nothing in `timeline.tsx` (token port
  complete).
- Manual (Playwright): docs `/` renders the Timeline; expand/collapse animates;
  the connector is one continuous line with no seams (the original fix holds in
  the new token set), in both light and dark themes.
- Smoke test the install in a throwaway app:
  `npx shadcn@latest add http://localhost:3000/r/timeline.json` (or deployed URL)
  pulls the component and it compiles.
